 E-Commerce Price Monitoring & Analytics Pipeline

 Project Overview

This project is an automated e-commerce pricing analytics pipeline designed to monitor product-level pricing trends and generate structured business insights.

It processes structured product data, performs data cleaning and transformation, and visualizes key pricing metrics through an interactive Streamlit dashboard.

The system demonstrates a complete analytics workflow:

Data Collection → Data Cleaning → Analysis → Visualization → Automation


 Objectives

- Monitor product pricing trends across categories
- Analyze average price distribution
- Track key performance metrics including:
  - Total Products
  - Average Price
  - Maximum Price
- Enable automated and scheduled data updates
- Provide business-ready visual insights


 Tech Stack

- Python
- Pandas
- Streamlit
- GitHub Actions (Automation)
- Matplotlib
- CSV Data Processing


 Workflow Architecture

1. Raw data ingestion  
2. Data cleaning and preprocessing using Pandas  
3. Exploratory Data Analysis (EDA)  
4. KPI computation  
5. Dashboard visualization using Streamlit  
6. Scheduled automation using GitHub Actions  


 Key Dashboard Features

- Real-time pricing metrics
- Category-wise price distribution
- Clean and structured UI
- Automatically updated dataset


 How to Run the Project

1. Clone the repository:

   git clone https://github.com/krithic616/ecommerce-price-pipeline.git

2. Navigate to the project folder:

   cd ecommerce-price-pipeline

3. Install dependencies:

   pip install -r requirements.txt

4. Run the Streamlit application:

   streamlit run app.py


 Business Context

This project simulates a real-world retail analytics use case where pricing trends and category-level insights support better pricing strategy, inventory planning, and business decision-making.

It demonstrates practical data analytics capabilities including automation, KPI reporting, data transformation, and interactive visualization.
